# מדריך למידה: MySQL עם JavaScript (Node.js)

ברוכים הבאים לערכת התרגול והלמידה של עבודה עם מסד הנתונים **MySQL** באמצעות **JavaScript (Node.js)**. 
כאן תלמדו כיצד להתחבר למסד נתונים, לבצע שאילתות, לאבטח אותן מפני פריצות, לנהל מאגרי חיבורים ולעבוד עם טרנזקציות.

---

## 🛠️ שלבי הכנה והרצה

כדי להתחיל לעבוד, בצעו את השלבים הבאים:

### שלב 1: הרמת מסד הנתונים ב-Docker
בתיקייה זו ישנו קובץ `docker-compose.yml` המגדיר שרת MySQL וממשק ניהול גרפי בשם Adminer.
הריצו את הפקודה הבאה במסוף (Terminal) מתוך תיקייה זו:
```bash
docker-compose up -d
```
> [!NOTE]
> פקודה זו תוריד ותריץ את השרתים ברקע. כעת שרת ה-MySQL שלכם מאזין בפורט `3306` עם הסיסמה `secret`.

### שלב 2: התקנת התלויות (Dependencies)
הריצו את הפקודה הבאה כדי להתקין את החבילות `mysql2` ו-`dotenv`:
```bash
npm install
```

### שלב 3: אתחול טבלאות ונתונים (Schema)
תוכלו להתחבר לממשק הניהול הגרפי **Adminer** בדפדפן בכתובת: [http://localhost:8080](http://localhost:8080).
- **System**: MySQL
- **Server**: `mysql` (במידה ואתם מתחברים מתוך קונטיינר אחר) או `localhost` (אם אתם מתחברים מהמחשב המקומי שלכם לפורט 3306)
- **Username**: `root`
- **Password**: `secret`
- **Database**: `learning_db`

כדי להריץ את קובץ ה-SQL שמייצר את הטבלאות, תוכלו להעתיק את תכולת הקובץ [schema.sql](file:///C:/Users/yaako/OneDrive/Desktop/study-kodcode/js-stading/week14/mysql/schema.sql) ולהריץ אותה בתוך ה-SQL Command ב-Adminer, או להריץ דרך ה-CLI אם מותקן לכם קליינט.

## 📂 מבנה התיקייה והשיעורים

התרגילים מסודרים בצורה פרוגרסיבית (שלב אחר שלב):

1. **`01_connection.js`** - חיבור בסיסי למסד הנתונים באמצעות Callback API.
   * *הרצה:* `npm run connection`
2. **`02_promise_connection.js`** - מעבר לעבודה מודרנית עם Promises ו-`async/await`.
   * *הרצה:* `npm run promise`
3. **`03_crud.js`** - ביצוע פעולות בסיסיות של כתיבה, קריאה, עדכון ומחיקה (CRUD).
   * *הרצה:* `npm run crud`
4. **`04_prepared_statements.js`** - אבטחת שאילתות מפני הזרקת קוד SQL (SQL Injection) בעזרת Placeholders (`?`) והשימוש ב-`connection.execute`.
   * *הרצה:* `npm run prepared`
5. **`05_connection_pool.js`** - עבודה נכונה באפליקציות שרת באמצעות מאגר חיבורים (Pool) לחיסכון במשאבים ושיפור ביצועים.
   * *הרצה:* `npm run pool`
6. **`06_transactions.js`** - ביצוע מספר פעולות התלויות אחת בשנייה כשלב אחד בלתי נפרד (Commit / Rollback) עם נעילת שורות במלאי.
   * *הרצה:* `npm run transactions`

## ✍️ תרגילי אתגר עבורך לתרגול עצמי

כדי להטמיע את החומר הנלמד, מומלץ לכתוב קבצים חדשים ולפתור את האתגרים הבאים:

### תרגיל 1: שליפת אימיילים בלבד (`exercise_emails.js`)
עליך ליצור קובץ חדש המשתמש ב-Connection Pool:
1. שולף את כל המשתמשים מטבלת `users`.
2. מדפיס למסוף אך ורק את רשימת האימיילים שלהם בצורת רשימה פשוטה.
3. אל תשכח לסגור את ה-Pool בסיום הריצה.

### תרגיל 2: הוספת טבלת הזמנות (`exercise_orders.js`)
עליך להוסיף טבלה חדשה בשם `orders` (הזמנות) המכילה:
- `id` (מפתח ראשי מוגדל אוטומטית)
- `user_id` (מזהה המשתמש שביצע את ההזמנה)
- `product_id` (מזהה המוצר שהוזמן)
- `quantity` (כמות שהוזמנה)
- `order_date` (תאריך ביצוע ההזמנה - ברירת מחדל הזמן הנוכחי)

כתוב קוד JavaScript בתוך טרנזקציה שמבצע הזמנה:
1. בודק אם יש מספיק מלאי למוצר המבוקש.
2. מוריד מהמלאי את הכמות המבוקשת (למשל 2 יחידות).
3. מוסיף שורה לטבלת `orders` עם מזהה המשתמש והמוצר.
4. במידה ואחד השלבים נכשל (למשל המלאי נמוך מהכמות המבוקשת) - בצע Rollback.
